# 查询工单编号和新工单编号接口 - 实现计划

## 需求概述
开发一个新的GET接口，用来查询工单编号和新工单编号，只返回有新工单编号的数据。

## 需求细节
1. **接口类型**：GET请求
2. **返回格式**：
   ```json
   {
     "success": true,
     "data": [
       {"orderid": "工单编号", "neworderid": "新工单编号"},
       ...
     ],
     "message": ""
   }
   ```
3. **过滤条件**：只返回有新工单编号（new_orderid不为空）的数据
4. **接口文档**：需要写入API文档.md

## 实现步骤

### [ ] 步骤1：添加数据库函数
- **文件**：database.py
- **函数**：`get_work_orders_with_new_orderid()`
- **功能**：查询所有有新工单编号的数据，返回orderid和new_orderid

### [ ] 步骤2：添加后端API接口
- **文件**：app.py
- **接口**：`GET /api/work-orders/new-orderids`
- **功能**：调用数据库函数返回数据
- **鉴权**：无需鉴权（第三方调用）

### [ ] 步骤3：更新API文档
- **文件**：API文档.md
- **添加**：新接口的完整说明

### [ ] 步骤4：测试接口
- 创建测试脚本验证功能

## 技术细节
- 数据库查询：`SELECT orderid, new_orderid FROM work_orders WHERE new_orderid IS NOT NULL AND new_orderid != ''`
- 返回字段：orderid（工单编号）、neworderid（新工单编号，注意字段名是小写的neworderid）

## 注意事项
1. neworderid字段名要使用小写
2. 只返回有新工单编号的数据
3. 接口无需鉴权
