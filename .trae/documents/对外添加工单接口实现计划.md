# 对外添加工单接口实现计划

## 需求概述

提供对外的 POST 接口，供第三方上传工单数据。数据格式与查询接口返回的 order.txt 格式一致。

## 确认的需求

1. **工单号重复处理**：如果 orderid 已存在，**拒绝**并返回错误
2. **附件图片处理**：

   * 如果 imageFilePath 是完整 HTTP 地址，则下载到本地存储

   * 否则存储为空字符串

## 实现步骤

### 步骤1：创建数据库函数 - 检查工单是否存在

**文件**: database.py
**函数**: `work_order_exists(orderid)`
**功能**: 检查工单号是否已存在于数据库

### 步骤2：创建数据库函数 - 下载远程图片到本地

**文件**: database.py
**函数**: `download_image(url)`
**功能**:

* 如果 url 是完整的 HTTP/HTTPS 地址，下载图片到 uploads 目录

* 否则返回空字符串

### 步骤3：创建数据库函数 - 解析上传数据格式并创建工单

**文件**: database.py
**函数**: `create_work_order_from_upload(order_data)`
**功能**:

* 解析扁平化的 key-value 格式数据

* 根据 path 字段判断数据归属到哪个表

* 正确提取和存储各字段数据

* 调用 download\_image 处理附件

### 步骤4：新增 API 接口

**文件**: app.py
**接口**: `POST /api/work-orders/upload`
**功能**:

* 接收第三方上传的工单数据（JSON格式）

* 检查工单号是否已存在（存在则返回错误）

* 调用 create\_work\_order\_from\_upload 创建工单

* 返回创建结果

* **注意：这是新增接口，不修改任何现有方法**

### 步骤5：测试

**文件**: test\_upload.py
**内容**:

* 使用 第三方上传数据格式.json 中的测试数据进行测试

* 验证数据正确存储

* 测试通过后更新 API 文档

### 步骤6：更新 API 文档

**文件**: API文档.md
**内容**:

* 添加上传接口说明

* 请求格式、响应格式、错误码说明

## 注意事项

1. 所有数组类型的字段都支持多条数据
2. 操作记录的 lastModifiedDate 和 content 使用相同索引
3. 附件图片只下载 HTTP 路径，非 HTTP 路径存储为空
4. 必须先检查 orderid 是否已存在，存在则返回错误
